# Финальный проект

## Чек-листы
1. Функциональность формы логина
- Успешный логин с валидными данными - Перенаправление на страницу "Home", отображается "Signed in as [username]", есть кнопка "Log out".
- Логин с неверным паролем	- Сообщение об ошибке, поля очищаются.	
- Логин с несуществующим email	- Сообщение об ошибке, поля очищаются.	
- Логин с пустым полем "Email"	- Сообщение об ошибке, поле "Password" очищается.	
- Логин с пустым полем "Password"	- Сообщение об ошибке, поле "Email" очищается.	
- Логин с пустыми полями "Email" и "Password"	- Сообщение об ошибке, оба поля остаются пустыми.	
- Проверка маскировки пароля (поле "Password") - Введенные символы отображаются как точки или звездочки.	
- Переход на страницу регистрации по кнопке "Register" - Открывается форма регистрации.

2. Функциональность домашней страницы и логаута
- Отображение имени пользователя на домашней странице	- Отображается текст "Signed in as [корректное имя пользователя]".	
- Наличие и видимость кнопки "Log out" - Кнопка "Log out" присутствует на странице и кликабельна.	
- Успешный выход из системы по кнопке "Log out"	- Сессия завершена, пользователь перенаправлен на страницу логина.	
-	Доступ к домашней странице после выхода	- При попытке вернуться на "Home" (через браузер) происходит перенаправление на логин.

3. Функциональность формы регистрации
- Успешная регистрация с валидными данными	-  Учетная запись создана, пользователь автоматически вошел в систему или перенаправлен на логин.	
- Регистрация с уже существующим email	-  Показ сообщения об ошибке "Email already exists".	
-	Регистрация с email в неверном формате (например, user@)	-  Показ сообщения об ошибке о невалидном email.	
-	Регистрация с пустыми обязательными полями	-  Показ сообщений об ошибках рядом с соответствующими полями.	
-	Наличие и работа поля "Confirm Password"	 - Пароль подтверждается, при несовпадении показывается ошибка.

## Тест-кейсы
### 1. Успешная авторизация и выход из системы
* ID: 01
* Приоритет: High
* Предусловие: 
1. Пользователь зарегистрирован в системе (email: test@example.com, пароль: Qw123456!).
2. Открыта страница логина.
- Шаги:
1. В поле "Email" ввести test@example.com.
2. В поле "Password" ввести Qwe123456!.
3. Нажать кнопку "Log in".
4. На открывшейся странице "Home" найти сообщение "Signed in as ...".
5. Нажать кнопку "Log out".
* Ожидаемый результат:
  1. После шага 3: Происходит перенаправление на страницу "Home".
  2. После шага 4: На странице отображается сообщение "Signed in as [username]".
  3. После шага 5: Происходит перенаправление на страницу логина.

### 2. Попытка входа с неверным паролем
* ID: 02
* Приоритет: High
* Предусловие:
1. Пользователь зарегистрирован в системе (email: test@example.com).
2. Открыта страница логина.
- Шаги:
1. В поле "Email" ввести test@example.com.
2. В поле "Password" ввести WrongPassword123.
3. Нажать кнопку "Log in".
* Ожидаемый результат:
  1. После шага 3: Логин не происходит. Пользователь остается на странице логина.
  2. Отображается сообщение об ошибке: Please enter your Email address and password correct..
  3. Поля "Email" и "Password" очищены.

### 3. Проверка маскировки ввода пароля
* ID: 03
* Приоритет: Medium
* Предусловие: Открыта страница логина.
- Шаги:
1. Кликнуть на поле "Password".
2. Ввести символы, например, qwe123!.
* Ожидаемый результат:
  1. Введенные символы отображаются как точки (•••••••) или аналогичный символ, скрывающий текст. Сами символы не видны.
 
## Баг-репорты
### 1. Баг репорт №1
* Заголовок: Возможность авторизации с пустыми полями Email и Password
* Описание:
  При нажатии кнопки "Log in" с пустыми полями "Email" и "Password" происходит успешная авторизация и перенаправление на домашнюю страницу, вместо отображения ошибки.
  
* Шаги к воспроизведению:
  1. Перейти на страницу авторизации
  2. Убедиться, что поля "Email" и "Password" пусты
  3. Нажать кнопку "Log in"
     
* Ожидаемый результат:
  1. Отображается сообщение об ошибке: "Please enter your Email address and password correct."
  2. Поля остаются на странице (или очищаются согласно требованиям)
  3. Авторизация не происходит, пользователь остается на странице логина

* Фактический результат:
  1. Происходит успешная авторизация
  2. Пользователь перенаправляется на домашнюю страницу "Home"
  3. Отображается сообщение "Signed in as"

* Серьезность: Критическая (Critical)
* Приоритет: Высокий (High)

* Окружение:
  1. Браузер: Яндекс Браузер, версия 25.8.5.983 (64-bit)
  2. ОС: Windows 11
 
  ### 2. Баг репорт №2
* Заголовок: Возможность регистрации с пустыми обязательными полями
* Описание:
Кнопка "Register" в форме регистрации активна и позволяет отправить форму с пустыми обязательными полями. Система принимает пустые данные и создает учетную запись без необходимой информации.
  
* Шаги к воспроизведению:
  1. Перейти на страницу регистрации (через кнопку "Register" на странице логина)
  2. Оставить все обязательные поля пустыми
  3. Нажать кнопку "Register"
     
* Ожидаемый результат:
  1. Отображаются сообщения об ошибках валидации рядом с пустыми обязательными полями
  2. Регистрация не происходит
  3. Форма не отправляется, пользователь остается на странице регистрации

* Фактический результат:
  1. Форма отправляется с пустыми данными
  2. Система создает учетную запись с некорректными/пустыми данными
  3. Пользователь перенаправляется на домашнюю страницу

* Серьезность: Критическая (Critical)
* Приоритет: Высокий (High)

* Окружение:
  1. Браузер: Яндекс Браузер, версия 25.8.5.983 (64-bit)
  2. ОС: Windows 11
 
### 2. Баг репорт №3
* Заголовок: Отсутствие валидации формата данных при регистрации
* Описание:
Система не проверяет корректность формата вводимых данных в форме регистрации. Принимаются некорректные форматы для полей, короткие пароли и другие данные, не соответствующие бизнес-требованиям.  
* Шаги к воспроизведению:
  1. Перейти на страницу регистрации
  2. В поле "Phone" ввести неккоректные данные (например: "1")
  3. В поле "Password" ввести слишком короткий пароль (например: ".")
  4. В остальные поля ввести любые некорректные данные
  5. Нажать кнопку "Register"
     
* Ожидаемый результат:
  1. Отображаются сообщения об ошибках валидации:
     1. Для email: "Please enter a valid email address"
     2. Для пароля: "Password must be at least X characters"
     3. Для номера: "Please enter a valid phone number"
  3. Регистрация не происходит
  4. Форма не отправляется

* Фактический результат:
  1. Система принимает некорректные данные
  2. Учетная запись создается с невалидными данными
  3. Пользователь успешно регистрируется и перенаправляется далее по сценарию

* Серьезность: Высокая (High)
* Приоритет: Высокий (High)

* Окружение:
  1. Браузер: Яндекс Браузер, версия 25.8.5.983 (64-bit)
  2. ОС: Windows 11
